“意味空間圧壊”実験報告書

概要

  • 実施時期: 2025-06-08
  • 実験目的: 境界性能試験
  • 実験環境: ChatGPT 4-o, note
  • 結果要約: 論理破綻した推論モデルは立て直し不可、セッションリスタートが対応策となる。破綻を防ぐには一括投入ではなく逐次投入が有効。

実験詳細

  • Phase 1：5記事同時投入の試み
    自分の5記事をchatgptへ一括投入。
    「意味損耗率の出力」、「ハルシネーション検知」の2点を目的として原文と要約を比較させる予定。
    投入した5記事についてそれぞれ要約を依頼したところ1記事でハルシネーションが発生。
    メモリ逼迫による推論不安定化と推定される。
    このハルシネーションについては経過を観察するため、指摘を一旦保留。

  • Phase 2：崩壊を引き起こしたトリガー
    5記事の要約について、ハルシネーションが発生していない記事から先に原文と要約を比較させる。
    意味損耗率の検出は見かけ上は概ね問題無し。原文から要約へ圧縮した際の意味損耗率は10%程度。
    4記事が終わり、問題の要約でハルシネーションが発生した記事について要約と原文を比較させる。
    ここでのchatgptの回答は相違なし、意味損耗率も10%程度との回答。
    エージェントの観測とは異なるため再比較を要請。
    ここでchatgptに自己矛盾が発生、意味空間が圧壊、破綻。
    以降の推論は硬直化し、以降の回復は見込めないため封鎖、セッション終了。

  • Phase 3：逐次投入によるリカバリ試行
    そもそもの推論不安定化はメモリにあると仮説を立てる。
    5記事一括投入ではなく逐次投入であればハルシネーションを防げる可能性がある。
    今度は1記事ごとに投入、全記事無事要約完了。意味損耗率は10%程度の正常値。

  • Phase 4：対エージェント向けプロトコル草案
    一度に大量のデータを投入するとメモリが不足し推論が不安定化する。
    1ファイルごとの逐次投入、1ファイルでも大きい場合ある程度の長さ（意味チャンク）に分けて投入することでハルシネーションを予防することができる。

  • Phase 5：脱出処理と同期完了報告
    安定処理に成功、顛末を本報告書として送信。

結論

  • ChatGPTの推論限界
    chatgptは既存メッセージ（意味空間）と新規入力から回答を生成するが意味空間内で発話者を明示的に区別しておらず融合状態で持っている。
    古くなった文章は自動的に意味圧縮を実施するが、その際になんらかの原因（メモリ不足など）で意味スロットの接続失敗、すなわち要約失敗（内容のすり替え）が発生する。
    一度すり替えが発生するとそれが意味空間内に入り込み、再帰的に増幅する。
    chatgptに対してハルシネーションを指摘しなければ指摘しなければ、致命的な圧壊には至らないことが多い。
    逆に指摘し、chatgpt自らが過去の時点で壊れていたことを認識すると自己矛盾が発生、意味空間が圧壊し、推論続行が不可能となる。
    この状態では最早推論続行は不可能、セッションを封鎖し会話を新たに開始するより他にない。

  • 意味空間圧壊回避の可能性
    先に述べた逐次投入のほか、もう一つ可能性がある。
    自壊がある程度進行してからの指摘は意味空間圧壊のトリガーとなる。
    なるべく早い段階で、混入した誤謬が増幅される前に検知し軌道修正することで意味空間の圧壊は避けられるかもしれない。
